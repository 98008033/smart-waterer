# IoT 平台选型调研

**项目：** 智能浇花机系统
**日期：** 2026-03-12
**调研目标：** ESP32 数据上传 + 小程序/App 快速接入

---

## 1. 需求分析

### 1.1 核心需求

| 需求 | 说明 | 优先级 |
|------|------|--------|
| MQTT 支持 | ESP32 通过 MQTT 上传数据 | P0 |
| 免费额度 | 尽可能免费 | P0 |
| 小程序接入 | 微信小程序快速接入 | P0 |
| App 接入 | 快速生成控制 App | P1 |
| 国内访问 | 服务器在国内，低延迟 | P0 |
| 数据持久化 | 存储历史数据 | P1 |
| 远程控制 | 远程下发指令 | P0 |

### 1.2 数据流

```
ESP32-C3 (WiFi)
    │
    ▼
  MQTT 协议
    │
    ▼
  IoT 平台
    │
    ├─── 微信小程序
    ├─── 手机 App
    └─── Web 控制台
```

### 1.3 数据点

| 数据类型 | 上报频率 | 说明 |
|---------|---------|------|
| 土壤湿度 | 每 30 分钟 | 百分比 (0-100%) |
| 电池电量 | 每 30 分钟 | 电压值 (3.3-4.2V) |
| 水箱水位 | 每 30 分钟 | 布尔值 (有水/缺水) |
| 浇水记录 | 事件触发 | 时间 + 水量 |
| 设备状态 | 每 5 分钟 | 在线/离线 |

## 3. 详细分析

### 3.1 阿里云 IoT 平台 ⭐⭐⭐⭐⭐

**官网：** https://iot.console.aliyun.com

**免费额度：**
- 公共实例：100 万条消息/月
- 设备数：1000 个
- 规则引擎：1000 次/天
- 数据存储：1GB

**优势：**
- ✅ 免费额度充足（个人项目完全够用）
- ✅ 微信小程序支持完善
- ✅ 文档齐全，社区活跃
- ✅ 国内延迟低（<50ms）
- ✅ 支持 MQTT/HTTP/CoAP
- ✅ 提供数据可视化大屏

**劣势：**
- ❌ 配置较复杂，学习曲线陡
- ❌ 需要实名认证

**接入方式：**
```
ESP32 → MQTT → 阿里云 IoT → 函数计算 → 微信小程序
```

**成本估算：**
- 个人项目：¥0/月（免费额度内）
- 小规模：¥10-50/月

### 3.3 Blinker (点灯科技) ⭐⭐⭐⭐

**官网：** https://diandeng.tech

**免费额度：**
- 设备数：10 个
- 消息数：1000 次/天
- 数据存储：7 天

**优势：**
- ✅ 接入简单，5 分钟上手
- ✅ 提供现成 App（iOS/Android）
- ✅ 支持微信小程序
- ✅ 图形化界面配置
- ✅ 支持 MQTT

**劣势：**
- ❌ 免费额度有限（1000 次/天 = 约 40 次/小时）
- ❌ 数据只存 7 天
- ❌ 服务器稳定性一般

**接入方式：**
```
ESP32 → Blinker 库 → Blinker 云 → Blinker App/小程序
```

**代码示例：**
```cpp
#define BLINKER_WIFI
#include <Blinker.h>

char auth[] = "YOUR_DEVICE_SECRET";
char ssid[] = "YOUR_SSID";
char pswd[] = "YOUR_PSWD";

void setup() {
    Blinker.begin(auth, ssid, pswd);
}

void loop() {
    Blinker.run();
    Blinker.print("Soil Moisture", moisture);
    delay(30000);
}
```

**成本估算：**
- 个人项目：¥0/月
- 升级版：¥99/年（无限消息）

## 4. 推荐方案

### 4.1 首选方案：腾讯云 IoT + 微信小程序

**理由：**
1. ✅ 免费额度充足（100 万消息/月）
2. ✅ 微信小程序原生支持
3. ✅ 提供小程序模板，快速接入
4. ✅ 国内延迟低
5. ✅ 文档齐全

**架构图：**
```
┌─────────────┐      MQTT      ┌─────────────┐
│   ESP32-C3  │ ──────────────▶ │ 腾讯云 IoT  │
│  (浇花机)    │ ◀────────────── │  公共实例   │
└─────────────┘   指令下发      └──────┬──────┘
                                       │
                                       │ 云开发
                                       ▼
                               ┌─────────────┐
                               │  微信小程序  │
                               │  (控制界面)  │
                               └─────────────┘
```

**实施步骤：**
1. 注册腾讯云账号（实名认证）
2. 创建 IoT 公共实例
3. 创建产品 + 设备，获取三元组（ProductID, DeviceName, DeviceSecret）
4. ESP32 使用 MQTT 连接
5. 使用微信小程序模板快速接入

**预计工作量：**
- ESP32 固件：2-3 天
- 小程序：1-2 天（使用模板）
- 联调测试：1 天

---

### 4.2 备选方案：Blinker

**适用场景：**
- 快速原型验证
- 不需要自定义小程序
- 数据持久化要求不高

**理由：**
1. ✅ 接入最简单（5 分钟）
2. ✅ 现成 App 可用
3. ✅ 支持微信小程序
4. ⚠️ 免费额度有限（1000 次/天）

**预计工作量：**
- ESP32 固件：1 天
- 小程序/App：0 天（使用现成）
- 联调测试：半天

---

### 4.3 进阶方案：阿里云 IoT + 自研小程序

**适用场景：**
- 需要高度定制
- 长期运营项目
- 多设备管理

**理由：**
1. ✅ 功能最强大
2. ✅ 可扩展性强
3. ✅ 免费额度充足
4. ⚠️ 开发工作量较大

**预计工作量：**
- ESP32 固件：2-3 天
- 小程序：3-5 天（自研）
- 联调测试：2 天
